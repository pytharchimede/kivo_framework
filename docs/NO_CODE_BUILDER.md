# Vision No-Code / Low-Code KIVO

KIVO doit évoluer en deux couches :

1. **Framework développeur** : CLI, code PHP, vues, composants, routes, migrations, API.
2. **Interface graphique** : création de modules, champs, permissions, menus, workflows et API sans écrire le code à la main.

## Interface prévue

- Catalogue de modules natifs : RH, Utilisateurs, Permissions, Tickets, CRM, Finance, Logistique, Site.
- Constructeur d'entités : champs, types, validations, relations.
- Générateur de pages : formulaire, listing, détail, dashboard.
- Générateur d'API mobile : endpoints, resources JSON, permissions.
- Constructeur de workflow : statuts, transitions, validateurs, notifications.
- Constructeur de portail : cartes de modules, icônes, couleurs, ordre, droits.
- Gestion des templates : enterprise, transit, BTP, médical, école, CRM.

## Règle importante

Le no-code ne doit pas cacher le code. Il doit générer du code propre, lisible, versionnable et modifiable.
